# Security Policy

## Supported versions

Security fixes are applied on the current `main` branch.

## Reporting a vulnerability

Please do not open public issues for security problems.

Instead, report vulnerabilities privately to the maintainer through GitHub private contact or a GitHub security advisory if enabled for the repository. Include:

- a clear description of the issue
- affected files or flows
- reproduction steps or proof of concept
- impact assessment if known

You will receive an acknowledgment as soon as practical, and the report will be handled privately until there is a fix or mitigation plan.
